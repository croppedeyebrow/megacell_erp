# DB 구조 정리 계획

현재 ERP DB는 두 종류의 테이블을 함께 사용한다.

## 1. 원본 적재 테이블

엑셀 파일을 읽어서 그대로 적재한 조회용 테이블이다.

- `수주`
- `재고`
- `재고출납`
- `BOM`
- `자재재고`
- `발주현황`
- `배터리*`
- `AS관리`

이 테이블들은 엑셀 재동기화 시 다시 만들어질 수 있다. 따라서 사용자가 ERP에서 직접 수정하는 업무 상태를 이 테이블에 저장하지 않는다.

## 2. ERP 운영 테이블

ERP 화면에서 변경되는 데이터를 저장하는 진짜 운영 테이블이다.

- `users`: 로그인/회원/권한
- `erp_orders`: 수주 운영 원장
- `erp_order_change_logs`: 수주 변경 이력
- `erp_inventory_movements`: 입출고/재고 이동 이력
- `erp_documents`: 문서 생성 이력
- `erp_sync_state`: 원본 동기화 상태

## 수주 데이터 운영 원칙

`수주` 테이블은 엑셀 원본이고, `erp_orders`는 ERP 운영 원장이다.

엑셀 원본에서 최초 동기화할 때는 `erp_orders`에 값을 복사한다. 이후 ERP에서 출고 처리, 금액 수정, 출고일 수정 등을 하면 `erp_orders.erp_modified`가 켜지고, 원본 엑셀을 다시 적재해도 ERP 수정값을 덮어쓰지 않는다.

변경이 발생하면 `erp_order_change_logs`에 이전값/변경값/변경자/변경시각/사유를 남긴다.

## 다음 개발 순서

1. `erp_orders` 기반 수주/미출고 화면 전환
2. 출고 처리 액션 추가
3. 금액/출고일 수정 폼 추가
4. `erp_order_change_logs` 변경 이력 화면 추가
5. 재고 차감은 `erp_inventory_movements`와 연결
